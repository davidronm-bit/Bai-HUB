Epic 1: Play the Pattern Betting Game 
Epic Description 
Players should be able to participate in a casino-style pattern betting game where they predict patterns (odd/even, lucky numbers, ranges, etc.) and receive feedback if they win or lose. 
Feature 1.1: Select a Betting Pattern 
Feature Description 
Players must be able to choose a pattern to bet on before the game generates a result. User Stories 
US1.1.1 
As a player, I want to choose a betting pattern so that I can try to predict the game result. 
US1.1.2 
As a player, I want to see multiple betting options so that I can choose the pattern I prefer. 
US1.1.3 
As a player, I want the betting options to be clearly labeled so that I understand the rules before placing a bet. 
Example betting options: 
● Odd 
● Even 
● Lucky 7 
● High (16–30) 
● Low (1–15) 
Feature 1.2: Place a Bet
Feature Description 
Players must be able to submit their chosen pattern as their bet. 
User Stories 
US1.2.1 
As a player, I want to submit my bet so that the game can evaluate whether I win or lose. 
US1.2.2 
As a player, I want to confirm my bet before the game runs so that I avoid mistakes. 
US1.2.3 
As a player, I want the system to acknowledge my bet so that I know it has been accepted. 
Epic 2: Game Result Generation 
Epic Description 
The system should generate a random result that determines whether the player wins or loses. 
Feature 2.1: Generate Random Game Result 
Feature Description 
The system generates a random number between 1 and 30. 
User Stories 
US2.1.1 
As a player, I want the game to generate a random number so that the result is unpredictable. 
US2.1.2 
As a player, I want the result generation to happen quickly so that the game feels responsive. 
Feature 2.2: Evaluate the Player's Bet 
Feature Description 
The system evaluates the bet using conditional logic.
User Stories 
US2.2.1 
As a player, I want the system to compare my bet with the generated number so that I know if I win. 
US2.2.2 
As a player, I want the game logic to follow the pattern rules so that results are fair. Example rules: 
● If bet = Odd → number must be odd 
● If bet = Lucky 7 → number divisible by 7 
● If bet = High → number > 15 
Epic 3: Display Game Results 
Epic Description 
Players must be able to clearly see the result of their bet. 
Feature 3.1: Show Generated Number 
User Stories 
US3.1.1 
As a player, I want to see the generated number so that I understand how the result was determined. 
Feature 3.2: Display Win or Lose Result 
User Stories
US3.2.1 
As a player, I want to know immediately if I won or lost so that I understand the outcome. 
US3.2.2 
As a player, I want the result to be clearly displayed so that it is easy to read. Example result display: 
Your Bet: Odd 
Generated Number: 9 
Result: You Win! 
Epic 4: Improve Player Engagement 
Epic Description 
The system should make the game enjoyable and interactive for players. 
Feature 4.1: Multiple Game Rounds 
User Stories 
US4.1.1 
As a player, I want to play another round after finishing one game so that I can continue playing. 
Feature 4.2: Points or Score System 
User Stories 
US4.2.1 
As a player, I want to earn points when I win so that the game feels rewarding. 
US4.2.2 
As a player, I want to see my score so that I can track my performance. Example scoring:
● Correct pattern → +10 points 
● Lucky 7 → +20 points 
Epic 5: User-Friendly Interface 
Epic Description 
Players should interact with a simple and easy-to-use interface. 
Feature 5.1: Game Interface 
User Stories

US5.1.2
